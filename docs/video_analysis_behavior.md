# 视频分析行为说明

## 1. 目标

本文档说明当前视频分析链路的实际行为，避免前端、后端、算法和验收方对输出预期不一致。

## 2. 当前实现

当前视频分析链路分为两部分：

### 2.1 抽帧语义分析

由 `analyze_video()` 负责：

- 按 `sample_interval` 和 `max_frames` 抽取关键帧
- 对抽中的帧运行 YOLO
- 对抽中的帧调用 Qwen 做语义理解
- 生成 JSON 结果

输出特点：

- JSON 不是逐帧结果
- JSON 是“抽样帧分析快照”

### 2.2 逐帧标注视频导出

由 `create_annotated_video()` 负责：

- 读取原始视频全部帧
- 每一帧重新运行 YOLO
- 持续绘制目标框
- 使用最近一次语义快照补充步骤、PPE 和报警状态

输出特点：

- 目标框是逐帧持续可视化
- 不再只在抽样帧上闪现

## 3. 报警与文字显示策略

### 3.1 全程保留

全程持续显示：

- 目标框
- 框体标签
- 报警外边框

### 3.2 尾段汇总

大块文字面板与报警说明仅在视频最后一段显示。

当前策略：

- 末尾 `2~5` 秒显示汇总面板
- 由 `_summary_window_sec()` 根据视频总时长动态决定

这样做的目的：

- 不长期遮挡目标框
- 保留完整结果说明

## 4. PPE 与报警来源

PPE 状态由两部分融合：

1. YOLO 检测结果
2. Qwen 语义结果

报警键当前主要有：

- `missing_gloves`
- `missing_goggles`
- `missing_lab_coat`

说明：

- 护目镜当前更多依赖 Qwen 语义补充，不应被视作高置信纯检测结论

## 5. 当前输出文件

视频分析一次任务会生成：

- `analysis_<task_id>.json`
- `annotated_<task_id>.mp4`

参考样例：

- `outputs/video_analysis/demo_45s_source.mp4`
- `outputs/video_analysis/demo_45s_annotated_qwen_v4.mp4`

## 6. 已知边界

### 6.1 不是实时流式推理

当前是离线视频处理后生成标注结果，不是浏览器中边播边算的实时推理。

### 6.2 JSON 与视频语义粒度不同

- JSON：抽帧级
- 视频框：逐帧级

这属于设计行为，不是 bug。

### 6.3 Windows 中文路径兼容性

若直接对中文路径文件做 OpenCV 读取，可能出现兼容问题。  
建议通过统一上传目录处理。

## 7. 后续可继续演进的方向

1. 去掉框上方类别文字，仅保留纯框体
2. 引入轨迹跟踪，减少逐帧检测抖动
3. 做 WebSocket 流式可视化
4. 将 JSON 从抽帧扩展为逐帧轻量事件流
