# Key Action Acceptance Pipeline

This pipeline refreshes a session-level acceptance package from existing key
action artifacts. It is designed to stay useful before annotation and training
data are available.

## Run

```powershell
.\scripts\run_acceptance_pipeline.ps1 -SessionDir data\sessions\exp_20260424_165718_dual_view
```

For structural validation against real evidence artifacts:

```powershell
.\scripts\run_acceptance_pipeline.ps1 -SessionDir data\sessions\exp_20260424_165718_dual_view -Strict
```

For a no-video/no-ffmpeg smoke run:

```powershell
.\scripts\run_acceptance_pipeline.ps1 -SessionDir data\sessions\tmp_pipeline_dry_run -DryRun -Strict
```

## Human Confirmation Loop

The first run writes:

- `exports/review_bundle.json`
- `metadata/confirmation_decisions.template.jsonl`

After a reviewer fills each row with `approved`, `rejected`, or `needs_review`,
apply the decisions:

```powershell
.\scripts\run_acceptance_pipeline.ps1 `
  -SessionDir data\sessions\exp_20260424_165718_dual_view `
  -DecisionsFile data\sessions\exp_20260424_165718_dual_view\metadata\confirmation_decisions.template.jsonl `
  -ApplyDecisions
```

Applied decisions are written to:

- `metadata/confirmation_decisions.applied.jsonl`
- `metadata/step_review_history.jsonl`

## Outputs

The pipeline writes the following acceptance package:

- `metadata/artifact_validation_report.json`
- `metadata/process_quality_report.json`
- `metadata/capability_gap_report.normalized.json`
- `exports/review_bundle.json`
- `metadata/confirmation_decisions.template.jsonl`
- `evaluation/evaluation_manifest.json`
- `reports/formal_evaluation_report.json`
- `reports/query_acceptance_report.json`
- `reports/query_acceptance_report.md`
- `reports/boss_report.md`
- `reports/acceptance_snapshot.json`

## Strict Semantics

`-Strict` requires at least one source evidence JSONL artifact, such as:

- `model_observation_events.jsonl`
- `video_understanding.jsonl`
- `state_change_index.jsonl`
- `material_asset_catalog.jsonl`
- `micro_segments.jsonl`
- `segment_index.jsonl`
- `retrieval_index.jsonl`

Reports generated by this pipeline do not count as source evidence. Annotation
and model training gaps are recorded, but they are not blockers for this stage.
