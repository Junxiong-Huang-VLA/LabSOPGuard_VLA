# Confirmation Audit Backend Notes

Worker D update for T-09/T-13. Scope stays inside `src/key_action_indexer`; this does not add frontend, PTZ, camera orchestration, or LabSOPGuard coupling.

## Artifacts

- `metadata/human_confirmation_queue.jsonl`: existing queue remains available and CLI-compatible.
- `metadata/human_confirmation_decisions.jsonl`: existing latest-decision file remains available.
- `metadata/human_confirmation_review_bundle.json`: review bundle generated by `confirmation-queue`; each item includes `evidence_summary`, `keyframe_refs`, `clip_refs`, `segment_refs`, and `suggested_action`.
- `metadata/human_confirmation_audit_trail.jsonl`: append-only audit events generated by `confirmation-decision`; each event records reviewer, note, previous decision, before state, after state, evidence summary, and decision-time media refs.
- `metadata/human_confirmation_review_summary.json`: step/evidence recap for later UI or report consumption.

## Compatibility

- Existing CLI arguments are unchanged: `confirmation-queue` and `confirmation-decision` still work as before.
- Dry-run sessions can generate all confirmation/audit JSON and JSONL artifacts without real video files or ffmpeg.
- YOLO evidence refs are treated as inputs to review metadata and retrieval-oriented recap, not as final bounding-box render deliverables.
